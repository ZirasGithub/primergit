Comandos basicos git

comandos basicos en órden:

  1. git init
  2. git status (ver el estado de los cambios del proyecto)
  3. git add (git add . (cumple la función de guardar los cambios))
  4. git commit (git commit -m "Primer commit")
    5. git remote
  5. git push (git push -u origin main POR UNICA VEZ EN PRIMERA INSTANCIA, LUEGO SOLO SE USA "git push")
    6. git remote add (crea una conexion a un repositorio remoto) (git remote add origin *link del remoto*)

Primer prueba de commit en github. Practicando la mecanica

Escribiendo en base a experiencia, para poder subir un cambio a github se deben hacer los siguientes pasos:

  1. Una vez hecha la modificación, se debe GUARDAR el codigo (aparecerá la letra M de MODIFIED), para luego escribir GIT STATUS para chequear el estado del archivo.
  2. Generamos el "paquete" de modificación con git add .
  3. Se commitea para guardar el cambio en el archivo .git localmente, para posteriormente enviarlo al repositorio remoto
  4. Se usa git push para empujar ese cambio al repositorio linkeado
